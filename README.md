# Relazio Plugin Example

Reference IP lookup addon built with `@relazio/plugin-sdk`.

It demonstrates:

- signed multi-tenant transform requests;
- authenticated install and unregister flows;
- synchronous and asynchronous transforms;
- idempotent signed webhook delivery;
- encrypted persistent installation storage;
- bounded HTTP payloads and result validation.

The lookup functions return mock data. Replace them with real providers before
shipping an addon.

## Requirements

- Node.js 18 or newer;
- `@relazio/plugin-sdk` 0.3.0 or newer;
- a public HTTPS origin for production.

Until SDK 0.3.0 is published to npm, this repository pins the audited package
tarball in `vendor/`. Replace the file dependency with `^0.3.0` after the npm
release.

## Configuration

Create a local `.env` from `.env.example` and set independent random secrets:

```bash
openssl rand -hex 32
```

Required variables:

| Variable | Purpose |
| --- | --- |
| `PUBLIC_URL` | Public addon origin used in manifest endpoints |
| `ADDON_INSTALL_TOKEN` | Protects personalized manifests and registration |
| `ADDON_STORAGE_ENCRYPTION_KEY` | Encrypts installation records at rest |
| `ADDON_STORAGE_PATH` | Encrypted installation file location |

`ADDON_ADMIN_TOKEN` is optional. When absent, `/stats` remains hidden.

The storage included here is intended for one addon replica. SaaS deployments
with multiple replicas should implement the SDK `InstallationStorage` interface
using a shared transactional database and `RequestReplayStore` using Redis or a
database with an atomic insert.

## Local Development

```bash
npm install
npm run dev
```

Health check:

```bash
curl http://localhost:3001/health
```

Personalized manifest:

```bash
curl "http://localhost:3001/manifest.json?installToken=$ADDON_INSTALL_TOKEN"
```

Install the exact personalized manifest URL from the Relazio Plugins screen.
Do not put that URL in logs, screenshots, tickets, or public documentation.

Transform and unregister endpoints require the signed server-to-server headers
generated by Relazio. They are intentionally not callable with an unsigned curl
command.

## Container Deployment

The provided Compose file binds the addon to loopback so a reverse proxy can
terminate TLS:

```bash
docker compose up -d --build
```

The named volume preserves the encrypted installation registry across restarts.
Back up the volume and encryption key separately. Losing the key makes the
registry unrecoverable.

For a public deployment:

1. Put an HTTPS reverse proxy in front of port 3001.
2. Set `PUBLIC_URL` to that exact public origin.
3. Keep the install and storage keys in a secret manager.
4. Restrict ingress to Relazio platform egress where practical.
5. Use a shared installation store and replay store before scaling beyond one
   replica.

## Routes

| Route | Authentication |
| --- | --- |
| `GET /health` | Public, no sensitive data |
| `GET /manifest.json` | Personalized install token |
| `POST /register` | Bearer install token |
| `POST /unregister` | Workspace HMAC signature |
| `POST /lookup-ip` | Workspace HMAC signature |
| `POST /scan-ip` | Workspace HMAC signature |
| `GET /stats` | Admin bearer token, otherwise hidden |
